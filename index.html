Welcome to the documentation page for gloom! Gloom is a 3D rendering header made for C++, primarily intended for use by beginners transitioning from other simpler rendering tools such as pygame. Gloom is incredibly limiting because of its simple nature. If you are looking to learn OpenGL, DO NOT use this header.

Gloom was made using other headers which were not created by me. These include: glad (glad.h), GLFW (glfw3.h), GLM (glm.hpp), and assimp (Importer.hpp). I take no credit for the creation of these headers, and only used them in this project.

Reasons to NOT use gloom:
- You are experienced with OpenGL
- You want to create 2D games / tools
- You want your rendering tool to be very flexible and advanced

Reasons to use gloom:
- You are inexperienced with OpenGL, and want to learn but you need a bridging gap
- You want to create 3D games / tools
- You want your rendering tool to be very simple and easy to use

Gloom is a RENDERING HEADER, and not a game engine. It is, simply put, definitions for functions in glad, glfw, glm, assimp, and opengl headers.
